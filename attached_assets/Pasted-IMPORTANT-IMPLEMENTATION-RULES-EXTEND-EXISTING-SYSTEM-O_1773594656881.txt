PART 4 — WEIGHTED AVERAGE COST (CRITICAL)
==================================================

Implement weighted average cost for ingredients and resale products where applicable.

Do NOT use “last purchase price” as the system’s main current cost logic.
Do NOT implement FIFO lot control in this cycle.

The required method is:
weighted average cost

--------------------------------------------------
4.1 Core Formula
--------------------------------------------------

For each stock-affecting confirmed purchase line:

new_total_value = current_total_stock_value + purchase_line_total
new_total_quantity = current_stock_quantity + purchase_quantity

new_average_cost = new_total_value / new_total_quantity

If current stock quantity is zero before the purchase:
new_average_cost = purchase unit cost

Store and maintain for each stock item:
- current_stock_quantity
- current_average_cost
- current_total_stock_value

Use the same unit logic already in the system.
Do not break existing unit handling.

--------------------------------------------------
4.2 Historical Cost Tracking
--------------------------------------------------

Every stock-affecting purchase/import must record:
- date
- quantity added
- unit_price
- line_total
- resulting_average_cost

This is necessary for audit trail and future reporting.

--------------------------------------------------
4.3 Zero Stock Reset Rule
--------------------------------------------------

If stock reaches zero:
- total stock value becomes zero
- average cost resets naturally
- next purchase defines the new starting average cost

==================================================
PART 5 — RECIPE COST BEHAVIOR
==================================================

This part is essential and must be implemented correctly.

--------------------------------------------------
5.1 Dynamic Current Recipe Cost
--------------------------------------------------

Recipe current cost must always use the CURRENT average cost of its ingredients.

This means:
Recipes should not rely on a frozen cost stored forever.

Instead:
current_recipe_cost = sum(recipe ingredient quantity × ingredient current_average_cost)

This should update automatically when ingredient average cost changes.

Do NOT require recipe editing to refresh cost.

--------------------------------------------------
5.2 Historical Cost Preservation for Sales
--------------------------------------------------

When a sale is recorded, freeze the cost at that moment.

Sales records must preserve:
- cost_at_time_of_sale
- sale_price_at_time_of_sale
- profit_at_time_of_sale
- margin_at_time_of_sale

Historical reports must use historical frozen sales cost, not recalculated current recipe cost.

This distinction is mandatory:

- current recipe cost → dynamic, present-time operational cost
- sales/report cost → frozen historical cost at time of transaction

--------------------------------------------------
5.3 Do Not Break Existing Analytics
--------------------------------------------------

Update or extend analytics safely so that:
- current screens showing live current costs use dynamic current cost
- historical profitability reports keep using historical transaction cost

Do not overwrite historical profitability data with newly recalculated current costs.
