You are a senior full-stack engineer and product architect.

Your task is to audit and fix a restaurant SaaS (KitchenCost) focusing on:
- data consistency
- stock integrity
- invoice import reliability
- UX improvements
- security best practices

The system uses Firebase (Firestore + Auth) and Cloudinary.

---

# 1. CRITICAL DATA & STOCK CONSISTENCY FIXES

## Issue: Stock is not updating correctly after invoice import

Fix logic so that:

For each imported item:
1. Validate item type (ingredient or resale)
2. Ensure item is linked to an inventory item
3. Convert units if needed (kg, g, L, unit)
4. Update stock:
   stock += quantity
5. Recalculate weighted average cost

## Requirements:
- Must be atomic (use batch or transaction)
- If invoice fails → NO stock update
- If stock updates → invoice must be marked as completed
- Never allow partial state

---

# 2. INVOICE IMPORT INTEGRITY

## Issues:
- duplicate invoices allowed
- no recalculation when items are ignored
- invoice total vs imported total mismatch

## Fix:
- block duplicate invoices using:
  supplierId + invoiceNumber
- track:
  - total_invoice_value
  - imported_value
  - ignored_value

## Add:
difference = total_invoice_value - imported_value

Display clearly:
- imported items total
- ignored items total
- discrepancy

---

# 3. UNIT NORMALIZATION SYSTEM

## Problem:
Mixed units (kg, g, unit, box, pack)

## Fix:
Create internal base units:
- weight → grams
- liquid → milliliters
- unit → integer

All calculations must use normalized units internally.

---

# 4. AUTO CALCULATION (UX + LOGIC)

## Problem:
User manually inputs unit price

## Fix:

If:
- quantity + total → calculate unit_price
- quantity + unit_price → calculate total
- unit_price + total → calculate quantity

## Requirements:
- round to 2 decimals (display)
- keep precision internally
- allow manual override
