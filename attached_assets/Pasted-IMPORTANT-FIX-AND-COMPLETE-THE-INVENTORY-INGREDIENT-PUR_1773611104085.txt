==================================================
CORE BUSINESS LOGIC WE WANT
==================================================

The system must follow this logic:

1. Ingredientes = master catalog of kitchen ingredients
2. Produtos de Revenda = master catalog of ready-made resale products
3. Estoque = current inventory state based on stock movements
4. Compras / Notas = source of stock entries
5. Produção = consumes ingredient stock
6. Vendas = consumes stock or records sold items depending on current system rules
7. Recipe current cost = based on current average ingredient cost
8. Historical sales cost = frozen at time of sale

This logic must be implemented consistently.

==================================================
PART 1 — DEFINE THE ROLE OF EACH MODULE
==================================================

Ingrediente:
- is a master item used in recipes and stock
- stores the identity of a kitchen input
- example: tomate, arroz, azeite

Produto de Revenda:
- is a master item sold ready-made, not produced by recipe
- example: refrigerante, vinho, água

Estoque:
- is NOT a separate master catalog
- it is the current inventory state derived from items + stock movements
- shows quantity, average cost, stock value, minimum stock, and status

Compras / Notas:
- create stock entry movements
- may create new ingredient or resale product only with user confirmation
- update inventory and costs

This distinction must be respected in code and UI.

==================================================
PART 2 — WHAT HAPPENS WHEN AN INVOICE IS IMPORTED
==================================================

For each confirmed invoice line:

If line type = ingredient:
1. link to existing ingredient OR create new ingredient if user confirmed
2. create stock entry movement
3. increase ingredient stock quantity
4. update ingredient average cost using weighted average cost
5. update inventory value

If line type = resale_product:
1. link to existing resale product OR create new resale product if user confirmed
2. create stock entry movement
3. increase product stock quantity
4. update average cost if resale products use stock cost tracking
5. update inventory value

If line type = ignore:
- save line in invoice record only
- do not affect stock

If line type = unknown:
- block import until resolved or ignored

==================================================
PART 7 — WEIGHTED AVERAGE COST
==================================================

Use weighted average cost as the current cost method.

Formula:

new_total_value = current_total_stock_value + incoming_line_total
new_total_quantity = current_stock_quantity + incoming_quantity
new_average_cost = new_total_value / new_total_quantity

If current stock was zero:
new_average_cost = incoming unit cost

Track for each item:
- current_stock_quantity
- current_average_cost
- current_total_stock_value

Do NOT use last purchase price as the primary current cost logic.

==================================================
PART 8 — INVENTORY VALUE LOGIC
==================================================

Inventory total value must be calculated as:

sum(item current_stock_quantity × item current_average_cost)

This value must update automatically after:
- invoice import
- manual entry
- manual exit
- production consumption
- other valid stock changes

==================================================
PART 9 — RECIPE CURRENT COST VS HISTORICAL SALES COST
==================================================

This distinction is mandatory.

A. Current recipe cost
- dynamic
- always recalculated from current average ingredient cost

Formula:
current_recipe_cost = sum(recipe ingredient quantity × ingredient current_average_cost)

B. Historical sales cost
- frozen at moment of sale
- never overwritten by later stock changes

Sales records and profitability reports must preserve:
- cost_at_time_of_sale
- sale_price_at_time_of_sale
- profit_at_time_of_sale
- margin_at_time_of_sale

Do NOT break existing reports by replacing historical cost with current cost.
